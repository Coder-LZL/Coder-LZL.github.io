1.1 this.$nextTick()

this.$nextTick 是 Vue.js 提供的一个方法，用于在下次 DOM 更新循环结束之后执行延迟回调。在 Vue.js 中，当数据发生变化时，DOM 不会立即更新，而是等到 Vue 完成其更新周期后，才会进行 DOM 的重新渲染和修补。this.$nextTick 方法允许我们在 DOM 更新完成后执行某些操作，确保我们操作的是最新的 DOM 状态。

实现原理

Vue.js 在更新 DOM 时，会异步地将所有数据变化队列化，然后在下一个事件循环“tick”中，统一执行 DOM 更新。this.$nextTick 方法就是利用这个机制，在 DOM 更新完成后，执行回调函数。

用途

(1)确保 DOM 更新完成：在数据变化后，需要操作 DOM 元素时，使用 this.$nextTick 可以确保 DOM 已经更新。
(2)获取更新后的 DOM 状态：在数据变化后，需要获取更新后的 DOM 元素的状态或属性时，可以使用 this.$nextTick。
(3)避免闪烁：在数据变化后，需要确保某些样式或动画效果在 DOM 更新后生效时，可以使用 this.$nextTick

1.2 this.$refs、ref、$el

(1)this.$refs：在Vue组件中，$refs是一个对象，用于存储所有带有ref属性的子组件或DOM元素的引用。ref属性通常在模板中通过ref属性在Vue来定义。
(2) $el：这是Vue组件实例的一个属性，用于访问组件的根DOM元素。通过this.$refs.slider.$el，可以获取到slider组件的根DOM元素。

 举例
<template>
  <div>
    <button ref="slider">滑动按钮</button>
  </div>
</template>

<script>
export default {
  mounted() {
    const sliderButton = this.$refs.slider;
    console.log(sliderButton); // 输出按钮的DOM元素
  }
}
</script>

用途

通常是在需要直接操作DOM元素时使用。例如，你可能需要监听某个按钮的点击事件，或者需要动态改变某个元素的样式。通过获取到DOM元素的引用，可以直接使用原生的JavaScript方法来操作DOM。

注意事项

(1)模板中定义ref：在使用this.$refs之前，需要在模板中为相应的元素或组件定义ref属性。
(2)组件挂载后访问：this.$refs只能在组件挂载后访问，因为ref是在组件挂载时被Vue添加到$refs对象中的。
(3)避免过度使用：虽然this.$refs可以提供直接操作DOM的能力，但过度使用可能会导致代码难以维护和测试。尽量使用Vue的数据绑定和指令来处理DOM操作，只有在必要的时候才使用this.$refs。

1.3 setAttribute(数值名,属性值)

setAttribute() 方法用于设置指定元素的属性值。如果属性已存在，则更新其值；如果属性不存在，则创建该属性。

语法

element.setAttribute(name, value)

参数

  • name：要设置的属性名称。
  • value：要设置的属性值。

返回值

无。

示例

// 设置元素的属性
const element = document.getElementById('myElement');
element.setAttribute('class', 'myClass');

1.4 attr()-只能获取元素的属性值

 :deep(.el-slider__button-wrapper::after) {
        content: attr(data-value);
 }

在上面的代码中attr()的作用是什么?

(1)在CSS中，attr()函数用于获取元素的属性值，并将其作为内容插入到伪元素中。在上面代码示例中，attr(data-value)用于获取具有data-value属性的元素的值，并将其作为内容插入到.el-slider__button-wrapper::after伪元素中。
(2)但是需要注意的是，在上面代码中content能通过attr(data-value)获取得到值前提是.el-slider__button-wrapper的元素有该属性，比如
获取slider组件根元素，并获取具有.el-slider__button-wrapper类名的第一个元素
const sliderButton = this.$refs.slider.$el.querySelector('.el-slider__button-wrapper');
if (sliderButton) {
    sliderButton.setAttribute('_dataValue', this.replyForm.scoreThreshold);
}

1.5 伪元素::before和::after

伪元素::before和::after是CSS中用于在元素内容之前或之后插入内容的一种方式。它们可以用来添加装饰性内容，如图标、引号等，也可以用来实现一些视觉效果，如边框、阴影等。

语法

selector::before {
  /* CSS样式 */
}
selector::after {
  /* CSS样式 */
}

示例(其中content是必需的,如果没有那么是不会显示的)

/* 在元素内容之前添加内容 */
.element::before {
  content: "Before ";
  color: red;
}

3.Slider中tooltip常显问题解决方法

解决后的结果----重要:记得把:show-tooltip改为false(即show-tooltip="false"，不然就会有两个tooltip)

<el-form-item label="分数阈值:" prop="question">
    <!-- <el-slider v-model="replyForm.scoreThreshold" show-tooltip :step="0.01" :min="0.8" :max="1.0" class="slider-type">
    </el-slider> -->
    <el-slider
        v-model="replyForm.scoreThreshold"
        :show-tooltip="false"
        :step="0.01"
        :min="0.8"
        :max="1.0"
        class="slider-type"
        ref="slider"
        @input="updateTooltip"
    />
    <p class="fraction-tip"><span class="green">0.8·容易匹配</span> <span class="blue">1.0·精准匹配</span></p>
</el-form-item>
mounted() {
    //页面挂载后即调用
    this.updateTooltip()
},
methods: {
    //实时更新tooltip的值
    updateTooltip() {
        this.$nextTick(() => {
        //console.log('组件的根元素',this.$refs.slider.$el);
        //获取.el-slider__button-wrapper类名的元素
        const sliderButton = this.$refs.slider.$el.querySelector('.el-slider__button-wrapper');
        //设置这个元素的属性和属性值
        if (sliderButton) {
            sliderButton.setAttribute('_dataValue', this.replyForm.scoreThreshold);
        }
        });
    },
}
